# L2 service thought experiments

This will be a new OpenZiti service that connects L2-enabled (tap) tunnelers to each other as if the overlay
was an ethernet cable.

## binding

Tunnelers will bind the service automatically. 

## dialing

Tunnelers will initiate L2 service connections with an IPC command that takes the target/destination tunneler's
identity as an argument, e.g.:

    ziti-edge-tunnel l2-dial <IDENTITY>

The command will block until the dial completes, indicating success or failure. When successful, the tunneler
will receive locally generated L2 frames. Frames with EtherTypes that match the service configuration will
be delivered to the overlay.

for now this configuration will be done with dial / listen options in the l2 service config types.

## config types

### l2.intercept.v1

```json
{
  "ethtypes": [ "PROFINET", "..." ],
  "dialOptions": { "identity": "l2-host" }
}
```
### l2.host.v1

```json
{
  "bridgeIfs": [ "eth0", "enp0s5", "..." ],
  "listenOptions": { "identity": "$tunneler_id.name" }
}
```

## demo setup

### create or update the config types:

```bash
ziti edge create config-type l2.demo.intercept.v1 -f ./l2.intercept.v1.json
ziti edge create config-type l2.demo.host.v1 -f ./l2.host.v1.json
```

### create/update service

```bash
ziti edge create config l2.demo.intercept l2.demo.intercept.v1 '{ "ethTypes": [ "0x0800" ], "dialOptions": { "identity":"l2.demo.host" } }'
ziti edge create config l2.demo.host l2.demo.host.v1 '{ "listenOptions": { "identity": "l2.demo.host" } }'
```
```bash
ziti edge create service l2.demo -c l2.demo.intercept,l2.demo.host -a l2.demo-dialers,l2.demo-binders
```
